🌙 Light & Dark Mode — Tailwind CSS

💡 Gunakan class strategy di Tailwind. Toggle dengan menambah/hapus class dark di tag <html>. Preferensi tersimpan di localStorage dan cookie.
jstailwind.config.js
module.exports = {
  darkMode: 'class', // toggle with .dark class
  theme: {
    extend: {
      colors: {
        primary: {
          500: '#7c5cfc', // MariBaca purple
          600: '#6b49e8',
        },
        accent: {
          500: '#0fb8a0', // MariBaca teal
        }
      }
    }
  }
}
tsxhooks/useTheme.ts
export const useTheme = () => {
  const [theme, setTheme] = useState<'light' | 'dark'>(
    () => localStorage.getItem('theme') as any || 'dark'
  )

  const toggle = () => {
    const next = theme === 'dark' ? 'light' : 'dark'
    setTheme(next)
    localStorage.setItem('theme', next)
    document.documentElement.classList.toggle('dark')
  }
  return { theme, toggle }
}

Arsitektur Sistem
5-layer architecture · Separation of Concerns · Scalable dari MVP sampai production
↓   Request dari Browser / Mobile
🖥  Client Layer — Next.js 14 + PWA
App Router (SSR/SSG) TipTap Editor React Query Zustand Store NextAuth.js Tailwind CSS Service Worker qrcode.react Framer Motion
↕   HTTPS REST / WebSocket (Socket.io)
🌐  API Layer — Express.js + Middleware
JWT Auth Middleware Redis Rate Limiter Zod Validator Multer Upload CORS Config Error Handler Socket.io
↕
⚙  Service Layer — Business Logic
AuthService (JWT + bcrypt) NovelService ChapterService SearchService (Meilisearch sync) NotificationService StorageService (R2 upload) EmailService (Resend)
↕
🗄  Data Layer — Prisma ORM + Redis
PostgreSQL (primary DB) Prisma ORM Redis (cache + session) Meilisearch (full-text) Cloudflare R2 (files)
↕
🚀  Infrastructure — Docker + Cloud
Docker Compose (local) Vercel (frontend) Railway (backend) GitHub Actions (CI/CD) Cloudflare CDN Sentry (monitoring)

Prinsip Arsitektur

🔀
Controller ≠ Service
Controller hanya menangani HTTP request/response. Semua logika bisnis ada di Service layer. Controller tidak boleh langsung akses Prisma.
🔒
Auth di Middleware
JWT verification terjadi di middleware, bukan di controller. Route yang butuh auth cukup tambahkan requireAuth middleware.
📦
Cache-First
Novel populer, daftar genre, dan homepage data di-cache di Redis dengan TTL 5 menit. Kurangi beban PostgreSQL hingga 80%.

Pola Controller → Service (WAJIB)

typescriptcontrollers/novel.controller.ts ↔ services/novel.service.ts
// ✅ BENAR — controller hanya handle HTTP
export const getNovel = async (req: Request, res: Response, next: NextFunction) => {
  try {
    const novel = await novelService.findBySlug(req.params.slug)
    if (!novel) return res.status(404).json({ message: 'Novel tidak ditemukan' })
    return res.json({ status: 'success', data: novel })
  } catch (err) { next(err) }  // lempar ke errorHandler
}

// ✅ BENAR — service handle logika bisnis + DB
export const findBySlug = async (slug: string): Promise<Novel | null> => {
  return prisma.novel.findUnique({
    where: { slug },
    include: { author: true, genres: { include: { genre: true } } }
  })
}

// ❌ SALAH — jangan akses prisma langsung dari controller
const novel = await prisma.novel.findUnique({ where: { slug } }) // JANGAN
